ANIMATION SYSTEM:
-----------------
Framework: Framer Motion 12.23.26

Helper Function:
```typescript
function getAnimationVariants(animation?: {
  enabled: boolean;
  type: string;
  duration?: number;
  delay?: number
}) {
  if (!animation?.enabled) return {};

  const duration = (animation.duration || 600) / 1000;
  const delay = (animation.delay || 0) / 1000;

  const variants = {
    hidden: {},
    visible: {
      transition: { duration, delay, ease: 'easeOut' }
    }
  };

  // Apply transformations based on type
  // (fade, slide-up, slide-down, slide-left, slide-right, zoom)

  return variants;
}
```

Usage Pattern:
```typescript
const variants = getAnimationVariants(section.animation);
const hasAnimation = section.animation?.enabled;

<motion.div
  initial={hasAnimation ? "hidden" : false}
  whileInView={hasAnimation ? "visible" : undefined}
  viewport={{ once: true, margin: "-100px" }}
  variants={variants}
>
  {/* Section content */}
</motion.div>
```

IMAGE CROPPING SYSTEM:
----------------------
Library: react-easy-crop

Workflow:
1. User selects image file
2. FileReader converts to data URL
3. ImageCropperModal opens with data URL
4. User crops to desired ratio
5. onCropComplete returns cropped blob
6. Blob converted to File and uploaded to Supabase
7. Public URL saved to database

Aspect Ratios:
- Hero Profile: 1:1 (square, circular display)
- Hero Background: 16:9 (widescreen)
- About Profile: Free-form (user choice)
- Skill Icons: 1:1 (enforced square)
- Navbar Logo: 1:1 (square)
- Projects Logo: 1:1 (square)

IMAGE DELETION SYSTEM:
----------------------
Flow:
1. User clicks red X button on image preview
2. window.confirm() shows warning dialog
3. If confirmed, check isSupabaseStorageUrl()
4. If Supabase URL, call deleteImage()
5. Remove from editor state
6. Automatic cleanup on save removes orphans

Storage Buckets:
- portfolio-images/hero-profiles
- portfolio-images/hero-backgrounds
- portfolio-images/about-profiles
- portfolio-images/skill-icons
- portfolio-images/navbar-logos
- portfolio-images/project-logos
